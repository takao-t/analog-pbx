# SLICユニット

## これは何？

SLIC(Subscriber Loop Interface Circuit)モジュールを使用して電話機を制御するためのユニットです。SLIC部分が"SLICモジュール"という名称なので、回路を付加して使えるようにした本機は"ユニット"と呼称することにしました。

加入者回線(電話機)と電気的なインタフェースを取り、その信号をやり取りできるようにしたユニットです。

SLICユニットは接続する回線数分、必要となります。もし2回線で使いたいのであれば2個用意します。とはいえ2回線だけのためにPBXを作るのは面倒！という人は2回線専用のミニPBXを利用してください。

https://github.com/takao-t/Mini-pbx/

ただし、トーン(DTMF)検出を行いたい場合には本プロジェクトを使用する必要があります。

## ライセンス
大元のプロジェクトライセンスに準じます。好きに使ってかまいませんが、改変する場合には継承表示をしてください。ただし商用利用(製品化、キット化、講習会を含む)は禁止です。商用利用したい場合には別途ご相談ください。

## 回路

回路図はこのディレクトリにPDFで置いてあります。

基本的にはAG1171の標準的な回路です。これをPIC(16F18326)で制御し、マイコンから扱いやすいようにしています。

加えてローカル、つまり回線毎に必要となるトーン(発信音やビジー音)の生成をPICで行い、電話機に送り込んだり、電話機からの音を聞いてDTMFをDPに変換する処理もPICで行っています。

電話機(phone)側の回路は保護用でESD対策です。

SLICユニット自体は消費電力が大きい(max 500mA)ので、もしRaspberry Pi等を交換制御に使うのであれば、SLICユニットの電源は5Vから取るようにしてください。電源電圧とロジック電圧に差が出ますので、その場合にはレベル変換用の抵抗を実装可能に設計してあります。

仕様上で電話機側の配線は1kmまでとされていますが、1kmのテストは行っていません。加えて対雷保護回路や電源混触保護が入っていないので、屋外での引き回しは避けてください。

基板製造用データも置いてありますので、ご利用ください。

## 制御信号

基本的に入力信号はLow側がアクティブです。内部プルアップ(Weak-pullup)によりピンを放置すると動作しない方向になっています。

|信号名|方向|説明|
|-----|-----|-----|
|T1,T2|入力|トーンを生成し電話機に流す(聴かせる)|
| | |T1=H, T2=H : トーンオフ|
| | |T1=L, T2=H : ビジートーン(ツーツー)|
| | |T1=H, T2=L : リングバックトーン(プルプル)|
| | |T1=L, T2=L : コーリングトーン(ツー連続)|
|HO|出力|フック信号(デバウンス済). L=オンフック, H=オフフック|
|RI|入力|鳴動指示(電話機のベルを鳴らす)|
|RE|入力|回線リバース. H=Normal, L=Reverse|

回線制御は基本4線しか使わない(リバース制御は現状のPBXコアでは使っていない)ので4本(とGND)を配線してください。

## 使用上の注意

本ユニットの番号出力はすべてダイヤルパルスです。HO出力を監視し、カウントすることで番号が得られます。

トーン電話機(DTMF)を使用する場合には本ユニットがDTMFを解釈し、ダイヤルパルスに変換して送出します。このため普通のプッシュホンのように連続してダイヤルできないので、パルス出力が完了したら「ププ」という音が聞こえるようになっています。次の番号をダイヤルするのはこのププ音が聞こえてから行ってください。

## 制御信号(シリアル制御)

マイコン等との接続配線数を極力減らしたい場合のためにシリアルモードを設けています。UART-RXに信号を入力することで動作します。ただし、このモードはあまり"真面目に"やってないのでデバッグが甘いです。
Raspberry Pi等でBit bangingでシリアルを処理することを考えて速度は遅めの1200bpsにしています。

シリアル制御の動作はちょっとややこしいので注意してください。シリアル制御には通常動作とシリアル動作の2種類があります。

通常動作ではシリアルのRXにコマンドを送ると『投げっぱなし』で処理されます。この場合にはPBX側にシリアル送出用ピンとHO入力用ピンの2ピンが必要になります。コマンド(トーンの生成とか)はシリアルで送り、フック状態やダイヤルパルスはHO信号で処理することになります。

SLICユニットのRXピンはREピンと兼用しています。このためリバース信号(RE)は1200bpsのデータによって動作しないように『鈍く』処理されています。

シリアル動作(+SER)ではRX/TX両方が有効になりSLICユニットはアンサバック、状態出力を行うようになります。このときTX信号用にT1ピンを兼用していますので、シリアル動作時にはT1に何も繋がないようにしてください。

なお各コマンドの終端は改行(0x0d)で、エコーバックはありません。

## 接続方法

単に電話機のベルを鳴らしたいだけならSLICユニット単体でも遊べます。電話機、電源と接続し、RIピンをGNDに落とすと電話機のベルがなります。ついでにT1,T2ピンをGNDに落とすと電話機からトーンも聞こえます。

マイコン(PBX本体)との接続は上記の制御信号を各回線毎に接続します。実際の音声はAI/AO部分で、AIが音声入力(外部から電話機へ)でAOが音声出力(電話機から外部へ)なのでVoIP化するような場合には、ここからオーディオインタフェースに繋ぎます。

本プロジェクトのスイッチボード等、クロスポイントスイッチに繋ぐ場合で、クロスポイントスイッチが両電源駆動ではない場合には、C5をショートしておくことでAO出力はDCバイアスされた音声出力となります。

## ケイデンス制御

電話に必要なケイデンス制御はPICが行います。ベルの鳴動は日本式は1秒鳴動、2秒休止の繰り返しですが、RI信号をLowにしておくと、この鳴動パターンを繰り返します。同様にトーンのビジーやリングバックもPICがケイデンス制御を行うので、SLICユニットへの入力信号では意識する必要はなく固定しておいてかまいません。

ベル鳴動等のケイデンスは信号が切り替わった(Hi->Low)時点を起点としてスタートするようにしてありますので、もし短くベルを鳴らしたいような場合には Hi->Low->Hi を短くすれば、ベルは短く鳴ります。鳴り分けしたいような場合にはPBXコア側から制御できるというわけです。

## Dive Into.

少しだけ内容を説明しておきます。

このユニットではSLICモジュールを制御しているため、SLICモジュールと同じ基板上にPICマイコンが乗っています。このためSLICモジュールの音声信号に介入することで、トーン生成やDTMF検出を可能にしています。

トーンはPICで生成し、SLICモジュールのVin(オーディオ入力)に『横から』流し込んでいます。

DTMFはSLICモジュールのVout(オーディオ)出力を横から『聴く』ことで検出し、検出された数字をダイヤルパルスに変換して出力します。これによりPBX側がDTMFを聴く必要がなく、番号はダイヤルパルスの処理のみ考えればよくなります。

DTMFの検出はPIC(16F18326)のADCでサンプリングし、ゲーツェルアルゴリズム(Goertzel algorithm)で行っています。PICの性能が高いため、これだけの機能を盛り込んでもまだ能力は余っています。DTMF検出は手持ちの電話機ではうまく動作していますが、もし動作しない電話機を見つけたら教えてください。

DTMF検出まわりは少しトリッキーです。回路図をみるとわかりますが、ADC入力に対し隣のピンを抵抗を介して接続しています。これはなぜなのかというと、電話機からのDTMF信号が±1Vあたりなので、ADCによる変換時になるべく大きな範囲で変換させたいので、ADCのVrefとしてPICのFVRで約2Vを与えています。音声信号なのでDCバイアスしてADCに加えなくてはいけませんから、このバイアス値をVref 2VにしたDACを使い、DAC出力を1/2とすることで約1Vのバイアス電圧としています。これでADCはフルスケール近くで使えることになります。この方法はPICのCIPと、1/2にバイアスするのにはプログラム(1行だけ)で行っています。DCバイアスを調整したい場合にはプログラムで調整も可能です。

フック出力(HO)はデバウンス処理されていますが、デバウンスはコード内では行わず、これもPICのCIPであるCLCを使って行っています。CLCによるデバウンスはプログラムを一切必要とせずPICの処理能力を食わないので便利です。
